3.1 名词定义 非必填

本需求新增名词的统一口径说明

名词定义
测试计划 挂载在某一资源位下的独立验证任务。包含测试 UID 白名单、测试时间段、测试素材等配置。与线上定向信息解耦，用于运营在上线前验证资源位展示效果。
测试 UID 白名单 测试计划中指定的用户 ID 列表。仅白名单内的用户在计划有效时段内可看到测试素材，用于精准圈定测试人群。
测试时间段 测试计划的生效时间窗口。支持预设（10 分钟、24 小时）或自定义起止时间。超出时段后计划自动失效，不再参与匹配。
测试素材 测试计划绑定的展示内容（图片、文案、跳转等）。命中测试计划的用户看到测试素材，而非资源位当前线上素材。
线上定向信息 资源位正式投放的定向规则集合，包括人群包、排除包、定制人群、实验分组、上下线状态、平台版本等。仅对未命中任何测试计划的用户生效。
P0 匹配（测试计划层） 服务端资源位匹配的第一优先级。用户 UID 在某条「进行中」测试计划的白名单内，且当前时间在计划时段内，则命中并返回测试素材，跳过后续全部线上规则。
P1 匹配（线上定向层） 服务端资源位匹配的第二优先级。仅当用户未命中任何有效测试计划时触发，按现有线上定向规则判断是否展示及展示内容。
精准隔离 测试计划命中后，无视人群包、实验、上下线、版本等全部线上定向规则，确保测试 UID 在有效时段内一定能看到测试素材，测试结果可预期、可复现。
计划状态 测试计划的生命周期状态：待生效（未到开始时间）、进行中（参与 P0 匹配）、已结束（到达结束时间自动退出）、已手动终止（运营主动停止）、已删除（未开始前删除）。

关系说明

关系基数说明
资源位 — 测试计划1 : N一个资源位可挂载多条测试计划，支持多人并行测试
资源位 — 线上定向信息1 : 1每个资源位对应一套线上正式投放规则
测试计划 — 测试 UID1 : N一条计划可指定多个测试 UID
测试计划 — 测试素材1 : 1每条计划绑定一套独立测试素材
用户 — 资源位请求 — 测试计划M : N用户请求时，可能命中 0 或 1 条测试计划（P0）
用户 — 资源位请求 — 线上定向M : 1未命中测试计划时，走线上定向匹配（P1）

业务规则摘要

环节规则
配置测试 UID 不再写入「定向信息 → 测试账号」，统一通过测试计划管理
匹配优先级P0 测试计划 > P1 线上定向；P0 命中后短路全部线上规则
并行测试同一资源位可挂多条计划，通过 UID 白名单隔离，互不干扰
线上安全非测试 UID 永远不走测试素材，仅走 P1 线上规则
时效计划到期自动「已结束」，测试 UID 回归线上规则，无需运营手动干预
流程图仅画到业务层面：运营配置 → 后台保存 → 服务端匹配 → 用户看到结果。不涉及接口协议、缓存、数据库等技术细节。
