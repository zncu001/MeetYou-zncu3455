策略规则（v4）

策略层边界：只描述状态判断结果与执行流程，不描述用户点击了哪个按钮；UI 入口与策略解耦。
统一策略，状态分流：判断「是否关联 → 是否授权」，命中结果后执行 A / B / C。
策略 A · 增量：判断结果 = 未关联 → 执行关联顺带授权；成功后变为已关联 · 已授权。
策略 B · 存量：判断结果 = 已关联且未授权 → 执行收款中补授权；成功后变为达标。
策略 C · 达标：判断结果 = 已关联且已授权 → 执行免确认打款，微信直接入零钱。
增量 ↔ 存量：增量授权成功 → 达标；达标后换绑微信 B → 对新账户回到增量；微信侧解约 → 回到存量。
拒绝授权：明确失败态（未完成授权 / 未授权无法打款），不静默成功。

需求描述（结构化 · v4）

1. 产品目标

在微信提现链路落地「自动到账（免确认收款）」能力：对满足条件的用户，美柚发起打款后资金直接入微信零钱，用户无需每笔确认收款。

2. 核心概念

概念定义解决什么状态值
关联微信账户将用户微信身份记为美柚提现收款账户钱打给谁已关联 / 未关联
授权自动到账用户同意商家免确认收款钱怎么到账已授权 / 未授权
增量用户判断结果为未关联（状态标签）执行策略 A未关联
存量用户判断结果为已关联且未授权执行策略 B已关联 + 未授权
达标用户判断结果为已关联且已授权执行策略 C已关联 + 已授权

增量 / 存量不是永久人群划分，而是对当前状态的策略分类；状态变化后立即切换策略。

3. 决策逻辑（仅状态判断）

  • 判断①：是否已关联微信账户？否 → 判断结果命中增量 → 执行 策略 A
  • 判断②：是否已授权自动到账？否 → 判断结果命中存量 → 执行 策略 B；是 → 执行 策略 C

上述判断与用户从哪个页面进入无关；同一判断结果对应同一套执行流程。

4. 策略模块（判断结果 → 执行流程）

策略 A · 判断结果：未关联

  • 执行：美柚发起关联顺带授权 → 微信授权页 → 用户同意 → 保存关联 + 开通状态 → 同步展示
  • 异常：用户拒绝 → 未完成授权，不可免确认打款
  • 状态结果：未关联 → 已关联 · 已授权（达标）

策略 B · 判断结果：已关联 + 未授权

  • 执行：发起收款中授权 → 用户勾选免确认并确认收款 → 记录开通 → 完成本次打款/提现
  • 异常：用户拒绝 → 未授权，无法打款
  • 状态结果：存量 → 达标

策略 C · 判断结果：已关联 + 已授权

  • 执行：美柚服务端发起免确认打款 → 微信执行转账入零钱 → App 到账通知
  • 状态结果：达标态维持不变

5. 状态转化（策略关联）

  • 增量 → 达标：策略 A 授权成功
  • 存量 → 达标：策略 B 补授权成功
  • 达标 → 增量：用户换绑关联微信 B（对新账户重新走策略 A）
  • 达标 → 存量：用户在微信侧关闭自动到账授权（下次提现走策略 B）

6. 关键规则

  • R1：判断结果为未关联 → 必须执行策略 A，直至关联（及授权）完成或明确失败
  • R2：判断结果为已关联未授权 → 不可走策略 C；须执行策略 B 或返回失败态
  • R3：打款由美柚服务端发起，微信执行转账
  • R4：策略之间通过状态连通，不以「场景」隔离
  • R5：拒绝授权必须有明确失败态
