# 大促策略迭代 · 逐字稿

> `[空格]` = 按空格推进演示 · `[停 2s]` = 留给听众消化 · `[看屏幕]` = 指向演示画面

---

这半年我参与了三次大促，双十一、年货节和 618。今天这部分我不是逐个讲"每次活动做了什么"，而是想跟大家分享一个我在这三次大促中逐渐形成的工作方式——就是怎么把上一次的复盘，变成下一次的策略输入，让我们在时间紧、资源有限的情况下，判断能越来越准。

先从双十一开始。

`[空格 → 扭蛋机摇晃，弹出选大奖弹层]`

双十一扭蛋活动不是我负责的需求，但活动结束之后我做了深度复盘。不是看结果好不好看，而是去看用户在整个活动链路里，到底做了什么、没做什么，行为数据在告诉我们什么。

`[看屏幕]`

大家看手机上这个弹窗。用户进入活动的第一步，是一个不可关闭的弹窗，要求先选一个大奖——SK-II 神仙水、1000 元现金、足金吊坠，三选一。选完之后才能开始抽奖。

`[空格 → 自动选中 1000 元现金，三个选项浮出真实选择占比，右侧弹出数据卡]`

60.26% 的用户选了 1000 元现金。

`[停 2s]`

SK-II 价值 1500，足金吊坠也值一千多——但超过六成的用户没选它们。这不是用户贪便宜，是用户在做一个信任判断：1000 元现金是三个选项里唯一一个不可能造假、不需要我去猜"到底是不是真的"的选项。

这个数据给我的启发是：在营销活动里，用户不是在选最贵的奖品，而是在选最确定的那个。利益点越直白、越不需要用户去"相信"，参与意愿就越高。

这个结论后来直接影响了年货节的设计——年货节我们就主打"抽签必得现金"，不搞实物大奖的选择了。

`[空格 → 用完抽奖次数，右侧弹出第二张洞察卡 89.93%]`

第二个发现。89.93% 的用户只参与了一天。

`[停 2s]`

人均参与天数只有 1.16 天。免费次数用完，用户就走了，第二天也不会再来。从参与次数的衰减曲线来看，3 次到 4 次有一个断崖式的下降——正好是免费次数用完的那个节点。

这说明什么呢？光靠给免费次数是留不住用户的。你给他 3 次免费，他抽完 3 次就走了。你需要一个机制，让他在用完免费机会之后，还有动力、也有路径继续参与。

`[停 2s，等整体成效卡和桥接文字出现]`

双十一整体带来了将近 4 万单增量，增长率 16.97%，ROI 1.26。这组数字本身不差。但对我来说，这些数字不是最重要的——**最重要的是这两个用户行为洞察**。

`[看屏幕右侧桥接文字]`

因为这两个洞察，直接变成了我设计年货节策略时的两根骨架。第一，用户要明确可信的利益点，那年货节就主打"抽签必得现金"，不搞含糊的。第二，用户参与深度不够，那年货节就引入任务体系，让用户在免费次数用完之后还有事可做。

`[空格 → 切换到年货节场景]`

`[停 2s，让场景切换完成]`

年货节是我主导策略设计的第一次大促。

但我没有从零开始想"年货节应该做个什么活动"。我做的事情是：基于双十一复盘的两个洞察，推导出三个具体的策略改动，然后用年货节的数据去验证这些改动是不是对的。

`[空格 → 第一次抽签，签条抽出，右侧弹出参与率对比卡]`

第一个改动：去掉"先选大奖"这个步骤。

`[看屏幕]`

大家注意，现在用户点"抽签得现金"就直接抽了，没有弹窗拦截，没有强制选择。

这不是一个简单的 UI 优化。双十一的数据告诉我，整体的奖品选择率只有 49.49%，也就是一半的用户看到那个弹窗之后，选都不选就离开了。而且首次进入活动就选大奖的只有 41.73%，但如果用户选了奖品之后，点击抽奖的比例高达 86.90%。

所以问题不是"用户不想参与"，而是"这个前置步骤在用户还不了解活动的时候就拦住了他"。对一个刚进来的用户来说，一个不可关闭的弹窗让他选奖品，他的第一反应不是"好开心"，而是"这是什么套路"。

`[停 2s]`

去掉之后呢？参与率从 42.95% 提升到了 48.34%，提升了 12.55 个百分点。仅仅是减少了一个多余动作。

这里面有一个我自己比较有感触的判断：不是所有的仪式感都是好的。仪式感应该放在用户已经建立信任之后，而不是在参与之前就设门槛。

`[空格 → 第二次抽签，右侧弹出参与深度对比卡]`

第二个改动：优化了首次抽奖的体验。

双十一的首抽是 100% 抽中淘宝超红，需要强制跳转到淘宝——用户第一次体验就是"被导去了另一个 App"，体验很差。复盘数据也验证了这一点：前 2 次抽中非营销类奖品的用户，后续人均参与次数反而比全都抽中营销类红包的用户高 3.15%，而且超红点击次数差不多。

所以年货节我把首抽改成了跟年味相关的即时奖励——上上签、坚果礼包、金豆这些，用户感受到"这个活动是真的在给我东西"，而不是"这是一个广告入口"。

结果是人均参与次数从 2.07 次提升到了 2.91 次，提升了 40.58%。用户首次参与后第二天还来的比例从 1.42% 到 2.72%，接近翻倍。

`[空格 → 第三次抽签，免费次数用完，任务面板从底部滑出]`

第三个改动，也是影响最大的一个。

`[看屏幕]`

大家看，免费次数用完之后，底部自动弹出了任务列表。领取淘宝年货红包、逛会场、浏览红包秘籍——这些都是低门槛的任务，完成一个就多得一次抽签机会。

这个设计背后的逻辑是这样的：用户在免费次数用完的那一刻，其实是他参与意愿最高的时候——他刚体验过了，知道活动是什么样的，也看到了自己可能得到什么。如果这时候你给他一个很简单的方式继续参与，他大概率会做。但如果什么都没有，他就走了，而且因为没有固定入口提醒，明天他也想不起来回来。

`[空格 → 点击任务，右侧弹出任务成效卡 + 整体成效卡]`

任务机制带来了 9,460 单增量，占整体活动增量的 12.52%。

12.52% 看起来不算特别大的一个占比。但它解决的是一个结构性问题：怎么把用户从"来了一次就走"变成"来了还愿意继续"。而且从任务偏好数据来看，点击率最高的是"领取淘宝年货红包"——9.60%。用户喜欢的是简单的、有明确好处的任务，不是复杂的浏览任务。这个结论后来也影响了 618 任务的设计。

`[停 2s]`

`[看屏幕右侧，等桥接文字出现]`

年货节整体的成效：新老用户合计增量 75,588 单，整体成单增长率 28.16%，对比双十一的 16.97%，提升了 65.94%。ROI 从 1.26 到 1.75。成单渗透率从 0.27% 提到了 0.31%。

但这些数字背后，我觉得更值得说的是：**它验证了"复盘驱动策略迭代"这个工作方式是有效的。**

我们不需要每次大促都从零开始想一个全新的创意。我们需要的是一个闭环——从上一次活动的数据里提炼问题，推导出策略方向，做出改动，再用下一次的数据去验证。双十一提供了问题，年货节提供了答案，而年货节的答案本身又会成为 618 的输入。

而且这种方式特别适合大促场景——因为大促的时间窗口很短，留给策划的时间很有限。如果每次都从零开始想，质量很难保证。但如果你有上一次的经验沉淀，你就能在有限的时间里做出更有把握的判断。

`[停 2s]`

接下来我会讲 618。先提前说一个数字：618 的整体参与率比年货节低了，但 ROI 从 1.75 跳到了 7.17。这说明我们的策略方向，已经开始从"想办法让更多人来参与"转向"让参与的人更精准地转化"了。
