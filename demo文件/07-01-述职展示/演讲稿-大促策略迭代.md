# 大促策略迭代 · 演讲稿

> 标注说明：`[空格]` = 按空格推进演示，`[停顿]` = 留 2-3 秒给听众消化

---

## 开场引入（30 秒）

这半年我参与了三次大促——双十一、年货节和 618。

但我今天不是要跟大家讲"我做了三场活动"，我想讲的是：**在大促节奏紧、资源有限的情况下，我是怎么把每一次活动变成下一次的策略输入，让我们的判断越来越准的。**

先从双十一说起。

---

## 第一幕：双十一扭蛋（约 2 分钟）

双十一的扭蛋活动不是我主导的需求。但活动结束后，我做了一件事——**深度复盘**。不是看结果好不好，而是看用户在活动链路里的行为，到底在告诉我们什么。

`[空格 → 扭蛋机摇晃，弹出大奖选择弹层]`

大家看，用户进入活动的第一步，是一个强制选择大奖的弹窗——SK-II、1000 元现金、足金吊坠，三选一。

`[空格 → 自动选中 1000 元现金，显示各选项占比，右侧弹出数据卡]`

**60.26% 的用户选了 1000 元现金。**

`[停顿]`

这个数据不是说用户贪便宜。SK-II 价值 1500、足金也值 1080——但用户不选。为什么？因为"1000 元现金"是三个选项里**唯一一个不可能造假、不需要信任成本的选项**。

这告诉我一件事：在营销活动里，用户不是在选"最贵的"，而是在选**"最确定的"**。利益点越明确、越可信，用户参与意愿越高。

`[空格 → 用完抽奖次数，右侧弹出第二张洞察卡]`

第二个发现：**89.93% 的用户只参与了一天。**

免费次数用完就走了，没有第二天再来的理由。人均参与天数只有 1.16 天。这说明什么？光靠"给免费次数"是留不住用户的——**你需要一个机制，让用户在用完免费机会之后，还有动力继续参与。**

`[停顿，等右侧整体成效卡和桥接文字出现]`

双十一整体带来了 39,630 单增量，增长率 16.97%，ROI 1.26。这组数据本身不差——但对我来说，**它更大的价值不是这些数字，而是这两个洞察。**

因为这两个洞察，直接成了我设计年货节策略的骨架：

- 用户要**明确可信的利益点** → 年货节主打"抽签必得现金"
- 用户**参与深度不足** → 年货节引入任务机制

`[空格 → 进入年货节场景]`

---

## 第二幕：年货节抽签（约 3 分钟）

年货节是我主导的第一次大促策略设计。

但我没有从零开始策划，而是**基于双十一的两个洞察，做了三个精确改动**，然后用数据验证这些改动是不是对的。

`[空格 → 第一次抽签，右侧弹出参与率数据卡]`

第一个改动：**去掉了"先选大奖"这个前置步骤。**

大家注意，现在用户点"抽签得现金"就能直接抽——没有弹窗拦截，没有强制选择。这不是一个 UI 优化，这是一个基于用户行为数据做的策略判断：双十一的奖品选择率只有 49.49%，说明这个步骤在赶走用户，不是在吸引用户。

结果呢？活动参与率从 42.95% 提升到 48.34%，**提升了 12.55 个百分点**——仅仅是减少了一个多余动作。

`[停顿]`

这里有一个我想强调的策略判断逻辑：**不是所有的"仪式感"都是好的**。当用户对活动还没有建立信任的时候，一个不可关闭的弹窗不是仪式感，是阻碍。仪式感应该在用户已经参与之后再给，而不是在参与之前就设门槛。

`[空格 → 第二次抽签，右侧弹出参与深度数据卡]`

第二个改动：**优化首次抽奖体验，首抽不再强制跳转电商平台。**

双十一首抽 100% 是淘宝超红，需要跳转出站——这让用户第一次体验就很差。年货节我改成了跟年味相关的实物奖品呈现，用户感受到"这个活动是真的在发奖"，而不是"这是一个广告跳转入口"。

人均参与次数从 2.07 提升到 2.91，**提升了 40.58%**。次日留存从 1.42% 到 2.72%，几乎翻倍。

`[空格 → 第三次抽签，免费次数用完，任务面板滑出]`

第三个改动，也是最重要的一个：**引入任务机制。**

大家看，免费次数用完之后，底部自动弹出了任务列表。用户可以通过完成简单任务——领红包、逛会场、浏览秘籍——来获得额外的抽签机会。

这个设计的策略逻辑是：**用户参与活动的意愿在免费次数用完的那一刻是最高的**——他刚体验过，知道这个活动是什么，也看到了奖品。如果这时候给他一个低门槛的方式继续参与，他大概率会做。但如果什么都不给，他就走了，明天也不会再来。

`[空格 → 点击任务，右侧弹出任务成效数据卡]`

任务机制带来了 **9,460 单**增量，占活动总增量的 **12.52%**。

这个数字说明什么？任务本身不是核心增长来源，但它起到了一个关键作用——**延长了用户的参与周期，让更多用户从"看了一眼"变成"真的下单了"。**

而且从任务偏好数据来看，点击率最高的是"领取淘宝年货红包"——9.60%。用户偏好**简单的、有明确利益点的任务**，而不是复杂的浏览任务。这个洞察后来也影响了 618 的任务设计。

`[停顿，等整体成效卡和桥接文字出现]`

整体来看年货节的成效：

- 新老用户合计增量 **75,588 单**
- 整体成单增长率 **28.16%**，对比双十一的 16.97%，**提升了 65.94%**
- ROI 从 1.26 提升到 **1.75**
- 成单渗透率从 0.27% 提升到 0.31%，**增长 16.75%**

`[停顿]`

但这些数字背后，我认为更重要的是它验证了一件事：

**基于复盘数据做出的策略判断是有效的。** 我们不需要每次大促都重新策划一个全新的活动——我们需要的是一个"复盘→洞察→改动→验证"的迭代机制。双十一提供了问题，年货节提供了答案，而这个答案本身又会成为 618 的输入。

## 过渡到 618（15 秒）

> 如果继续讲 618，可以用这段过渡：

年货节之后，我带着这些经验进入了 618。618 的数据我一会儿会展开讲——但先提前说一个关键结论：618 的整体参与率比年货节低了，但 **ROI 从 1.75 跳到了 7.17**。

这说明什么？说明我们从"想办法让更多人参与"，开始转向了**"让参与的人更精准地转化"**——这本身就是策略在迭代。

## 这段演讲暗示的能力信号

不需要直接说，但听的人会感受到：

1. **主动从非自己负责的项目中提取策略方向**（双十一不是我负责的，但我做了复盘）
2. **有可复用的分析框架**（DID 分析方法、"复盘→洞察→改动→验证"的迭代闭环）
3. **从数据中读出用户行为逻辑，而不只是看数字好不好看**（60.26% → 用户要确定性，不是用户贪便宜）
4. **知道什么时候做减法**（去掉选大奖步骤）比知道什么时候做加法更难
5. **能区分"指标好"和"策略对"**（双十一数据不差，但策略方向需要调整）
